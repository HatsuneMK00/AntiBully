# 需求分析

## 需要完成的模块

### 课程模块

#### 用例描述

用例名称：课程观看

参与者：用户

前置条件：无

后置条件：该视频课后练习题的正确率被记录

主成功场景：

1. 从主页点击功能按钮进入课程列表页
2. 选择观看的课程集数
3. 观看视频内容
4. 完成课程对应练习题

扩展场景：

4-a: 题目回答错误时显示原因并显示正确答案
4-b: 题目回答正确时显示原因

### 答题模块

#### 用例描述

用例名称：答题

参与者：用户

前置条件：无

后置条件：已完成的题库章节被记录，已完成的**题目编号**被记录，每个完成的题库正确率被记录

主成功场景：

1. 从主页点击功能按钮进入题库列表页（题库分类显示）（含有综合题）
2. 点选后返回10道属于该题库的题目
3. 进入题目页，每页显示1道题
4. 全部完成后返回题库列表页

扩展场景：

3-a: 没有选择答案的情况下不能进入下一题

### 求助模块

#### 用例描述

用例名称： 求助

参与者：用户

前置条件：用户完成登陆

后置条件：用户的求助信息上传到服务器

主成功场景：

1. 从主页点击功能按钮进入求助页面
2. 填写情况和联系方式
3. 提交表单

扩展场景：

1-a: 显示提示文本，比如咨询电话或友情提示

#### 用例描述

用例名称：填写问卷

参与者：用户
